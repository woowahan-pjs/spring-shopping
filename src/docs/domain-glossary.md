## 도메인 용어 사전 (Ubiquitous Language)
이 문서는 프로젝트에 참여하는 모든 구성원(기획자, 디자이너, 개발자)이 동일한 언어로 소통하기 위한 용어 사전입니다.
코드, 문서, 대화에서 아래 용어를 일관되게 사용합니다.

### 하위 도메인 및 바운디드 컨텍스트
| 바운디드 컨텍스트 | 책임 |
|---------|-------------|
| Member | 회원 가입 및 로그인 처리. 인증 토큰 발급 및 검증 | 
| Product | 상품 생명주기(등록/조회/수정/삭제) 관리. 비속어 필터링 | 
| Wish | 인증된 회원의 관심 상품 목록 관리 | 



### 회원

| **한글명** | **영문명**     | **설명**                                         |
|--------|-------------|------------------------------------------------|
| 회원   | Member      | 서비스에 가입한 사용자. 이메일과 비밀번호로 식별됨                       |
| 회원 ID  | id          | 내부 시스템에서 식별하는 회원의 고유 식별자                       |
| 이메일    | email       | 로그인 시 사용하는 식별자                                 |
| 비밀번호   | password    | 회원 인증을 위한 비밀번호 (암호화 저장)                        |
| 인증 토큰  | accessToken | 로그인 성공 시 발급되는 문자열로, 이후 보호된 API 접근 시 인증 수단으로 사용 |
- 정책
    - 이메일은 올바른 형식이어야 하며 중복 가입이 불가하다.
    - 비밀번호는 최소 8자, 최대 20자로 제한한다.
    - 인증 토큰은 Authorization: Bearer {token} 형식으로 전달하며, 위시 리스트 API 접근 시 필수다.


### 상품

| **한글명** | **영문명**  | **설명**                                                              |
|---------|----------|---------------------------------------------------------------------|
| 상품    | Product  | 판매 목적으로 등록된 항목. 이름, 가격, 이미지 URL로 구성됨                                                          |
| 상품 ID   | id       | 상품의 고유 식별자                                                          |
| 상품 이름   | name     | 상품의 명칭. 공백 포함 최대 15자이며, 허용된 특수문자((, ), [, ], +, -, &, /, _) 외 사용 불가 |
| 상품 가격   | price    | 상품의 판매 가격. 0원 이상 999,999,999원 이하                                    |
| 이미지 URL | imageUrl | 상품 이미지가 호스팅된 외부 주소 문자열. 파일 업로드 방식은 허용하지 않음                          |
- 정책
  - 상품 이름에는 허용된 특수문자((, ), [, ], +, -, &, /, _) 외 사용이 불가하다
  - 상품 등록/수정 시 비속어가 포함된 이름은 거부된다. (PurgoMalum API 검증)
  - 상품 이미지는 URL 입력 방식만 허용한다.

### 위시리스트

| **한글명** | **영문명**   | **설명**                    |
|---------|-----------|---------------------------|
| 위시리스트   | Wishlist  | 특정 회원이 관심 있는 상품을 모아둔 목록            |
| 위시 ID   | Id        | 위시리스트의 고유 식별자             |
| 회원 ID   | memberId  | 해당 위시 리스트 항목을 소유한 회원의 식별자 |
| 상품 ID   | productId | 위시 리스트에 담긴 대상 상품의 식별자     |
- 정책
  - 위시 리스트의 모든 기능(조회/추가/삭제)은 인증 토큰을 가진 회원만 사용할 수 있다.
  - 존재하지 않는 상품은 위시 리스트에 추가할 수 없다.
  - 위시 항목 삭제는 해당 항목을 추가한 본인만 가능하다.

### 도메인 간의 관계

Member (1) ──────< Wishlist >────── (1) Product
- 한 명의 회원은 n개 이상의 위시 항목을 가질 수 있다.
- 하나의 상품은 n개 이상의 위시 항목에 포함될 수 있다.
- Wishlist Member와 Product를 연결하는 중간 개념이다.
